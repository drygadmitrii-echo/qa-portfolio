# Яндекс.Маршруты (Web) — UI/вёрстка, окна оплаты и баг-репорты

Кейс по веб-интерфейсу: проверка вёрстки, модальных окон и сценариев вокруг оплаты/карты — плюс фиксация дефектов.

## Что сделано
- чек-лист по вёрстке: **117** проверок
- чек-лист по окнам оплаты/добавления карты: **39** проверок
- тест-кейсы на кнопку "Забронировать": **6**
- баг-репорты: **14**

## Файл
- routes_web_ui_checklists_cases_bugs.xlsx — все артефакты в одной таблице

## Небольшое личное
UI-тестирование учит терпению: иногда баг — это "кнопка на 2 пикселя уехала", но именно так у пользователя ломается ощущение качества.
Запомнилось, как "Каршеринг" показывал иконку "Драйв" — вроде мелочь, а доверие к интерфейсу сразу падает.

## Как открыть
Скачайте .xlsx и откройте в Excel/Google Sheets.
